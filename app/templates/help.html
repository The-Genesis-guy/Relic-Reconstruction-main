Help & Reference Guide

Complete reference for all terms, verdicts, and features used in the platform.

📋 Submission Verdicts

When you submit code, it goes through our judging system. Here are all possible verdicts:

Verdict Full Name Meaning Points
AC Accepted Your code passed all test cases! ✓ Full points
WA Wrong Answer Your code ran but produced incorrect output for some test cases. Partial points (based on tests passed)
TLE Time Limit Exceeded Your code took too long to execute (> 5 seconds per test). 0 points
RE Runtime Error Your code crashed during execution (e.g., division by zero, array out of bounds). 0 points
CE Compilation Error Your code has syntax errors and couldn't compile/run. 0 points
Pending Pending Your submission is in the queue waiting to be judged. -
💡 Tip: Even with WA (Wrong Answer), you can get partial credit! The more test cases you pass, the more points you earn.

📝 Problem Types

1. Coding Problems (Function Mode)

You implement a specific function. The platform provides starter code with the function signature.

Example:
def twoSum(nums, target):
You only need to complete the function body. Input/output handling is done automatically.

2. Debugging Problems (stdin/stdout Mode)

You read input from stdin and write output to stdout. Full program required.

Example:
n = int(input())
print(result)
You handle all input/output yourself.

🏆 Scoring System

How Points Are Calculated

  • Sample Tests: Visible test cases that give 0 points (for practice)
  • Hidden Tests: Secret test cases that determine your score
  • Partial Credit: Pass X out of Y tests = (X/Y) × Total Points
  • Best Submission: Only your highest score counts per problem
💡 Strategy: Test your code with sample tests first (Run button), then submit when confident!

Leaderboard Ranking

  1. Total Points: Sum of best scores across all problems
  2. Time Penalty: Earlier submissions rank higher in case of ties
  3. Problems Solved: Number of problems with points > 0

⏱️ Time Limits & Constraints

Limit Type Value Description
Execution Time 5 seconds per test Maximum time your code can run for each test case
Code Size 50 KB Maximum size of your submitted code
Output Size 10 KB Maximum output your code can produce
Run Cooldown 3 seconds Wait time between clicking "Run" button
Submit Cooldown 10 seconds Wait time between submissions
⚠️ Important: If your code exceeds the time limit, you'll get TLE (Time Limit Exceeded) verdict with 0 points.

🎮 Contest Features

Grace Period

You get an extra 30 seconds after the contest ends to submit your last solution. This helps if you're finishing up right at the deadline!

Real-Time Updates

The leaderboard and submission status update automatically using WebSocket technology. No need to refresh the page!

Multiple Attempts

You can submit as many times as you want (with 10-second cooldown). Only your best score per problem counts.

💡 Tips & Tricks

Before Submitting:

  • Test with sample inputs using the "Run" button
  • Check edge cases (empty input, maximum values, etc.)
  • Verify your output format matches exactly
  • Consider time complexity for large inputs

If You Get WA (Wrong Answer):

  • Check your logic carefully
  • Test with more examples
  • Look for off-by-one errors
  • Verify data types (int vs float)

If You Get TLE (Time Limit Exceeded):

  • Optimize your algorithm (reduce time complexity)
  • Avoid nested loops when possible
  • Use efficient data structures (hash maps, sets)
  • Remove unnecessary operations

If You Get RE (Runtime Error):

  • Check array/list bounds
  • Handle division by zero
  • Verify input parsing
  • Check for null/None values
